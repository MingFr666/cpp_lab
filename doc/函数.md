# C++ 函数体系总
## 1. 普通函数

普通函数是 C++ 中最基础的可调用实体。

```cpp
int add(int a, int b) {
    return a + b;
}

int x = add(1, 2);
```

普通函数编译后会变成一段机器指令，函数名在很多场景下可以退化为函数地址。

```cpp
int (*fp)(int, int) = add;
```

这里的 `add` 可以理解为函数入口地址。

---

## 2. 函数声明和函数定义

### 2.1 函数声明

```cpp
int add(int a, int b);
```

函数声明只是告诉编译器：

> 有一个叫 `add` 的函数，它接收两个 `int` 参数，返回一个 `int`。

声明本身不会生成函数体代码。

---

### 2.2 函数定义

```cpp
int add(int a, int b) {
    return a + b;
}
```

函数定义提供了真正的函数体，会生成对应的代码。

---

## 3. 函数重载

C++ 支持函数重载，即同名函数可以有不同参数列表。

```cpp
int add(int a, int b);
double add(double a, double b);
```

注意：**仅返回值不同不能构成重载**。

```cpp
int f(int);
double f(int);  // 错误：参数列表相同，仅返回值不同
```

---

## 4. 函数指针

函数指针是指向函数入口地址的指针。

```cpp
int add(int a, int b) {
    return a + b;
}

int (*fp)(int, int) = add;

int result = fp(1, 2);
```

### 4.1 函数指针语法拆解

```cpp
int (*fp)(int, int);
```

含义是：

```text
fp 是一个指针
它指向一个函数
这个函数接收两个 int 参数
返回值是 int
```

必须注意括号：

```cpp
int (*fp)(int, int);  // 函数指针
int *fp(int, int);    // 函数声明，返回 int*
```

---

### 4.2 使用 using 简化函数指针

```cpp
using FuncPtr = int (*)(int, int);

FuncPtr fp = add;
```

比 `typedef` 更直观。

---

### 4.3 函数指针的典型用途

#### 回调函数

```cpp
void process(int x, void (*callback)(int)) {
    callback(x);
}

void print(int x) {
    std::cout << x << std::endl;
}

process(10, print);
```

#### 策略切换

```cpp
int add(int a, int b) { return a + b; }
int sub(int a, int b) { return a - b; }

int calc(int a, int b, int (*op)(int, int)) {
    return op(a, b);
}

calc(10, 3, add);
calc(10, 3, sub);
```

---

### 4.4 函数指针的限制

函数指针可以指向：

```text
普通函数
static 成员函数
无捕获 lambda
```

不能直接指向：

```text
普通成员函数
有捕获 lambda
函数对象
```

例如普通成员函数：

```cpp
class A {
public:
    void f();
};
```

它的类型不是：

```cpp
void (*)();
```

而是：

```cpp
void (A::*)();
```

因为普通成员函数调用时还需要一个对象，也就是隐藏的 `this` 指针。

---

## 5. inline 函数

`inline` 有两个层面的含义：

```text
1. 语言层面：允许函数定义出现在多个翻译单元中
2. 优化层面：建议编译器内联展开函数调用
```

现代 C++ 中，`inline` 更重要的作用是第一个：**允许多处定义**。

---

### 5.1 inline 不保证真的展开

```cpp
inline int add(int a, int b) {
    return a + b;
}
```

写了 `inline`，并不代表编译器一定会把函数展开。

是否真的展开，取决于编译器优化。

影响因素包括：

```text
函数体大小
调用频率
优化等级：-O0 / -O2 / -O3
是否递归
是否跨翻译单元
是否开启 LTO
目标平台和 ABI
编译器自己的成本模型
```

所以：

```text
写了 inline，不一定展开
没写 inline，也可能被编译器展开
```

---

### 5.2 inline 的 ODR 作用

如果函数定义放在头文件里：

```cpp
// util.h
inline int add(int a, int b) {
    return a + b;
}
```

这个头文件被多个 `.cpp` 包含时，不会因为多份定义导致链接报错。

因为 `inline` 函数允许在多个翻译单元中有相同定义。

---

## 6. `.h` 里类成员函数和 inline 的关系

关键要区分：

```text
类中声明
类中定义
类外定义
```

---

### 6.1 类中只有声明：不是 inline

```cpp
// A.h
class A {
public:
    void f();   // 只是声明，不是 inline
};
```

实现通常放在 `.cpp` 里：

```cpp
// A.cpp
#include "A.h"

void A::f() {
    // ...
}
```

这种情况下，头文件可以被多个 `.cpp` 引用，因为头文件中只有声明，没有定义。

---

### 6.2 类中直接写函数体：隐式 inline

```cpp
// A.h
class A {
public:
    void f() {
        // ...
    }
};
```

这叫类内定义成员函数。

它隐式是 `inline` 的，等价于：

```cpp
class A {
public:
    inline void f() {
        // ...
    }
};
```

所以这种写法放在头文件中，被多个 `.cpp` 引用也不会重复定义报错。

---

### 6.3 类外定义放在头文件里：需要显式 inline

错误写法：

```cpp
// A.h
class A {
public:
    void f();
};

void A::f() {
    // ...
}
```

如果多个 `.cpp` 都包含这个头文件，链接时可能报：

```text
multiple definition of `A::f()`
```

正确写法：

```cpp
// A.h
class A {
public:
    void f();
};

inline void A::f() {
    // ...
}
```

或者把定义放进 `.cpp` 文件中。

---

### 6.4 总结规则

```text
类中只有声明：不是 inline，但不会重复定义
类中直接写函数体：隐式 inline
类外写函数体且放在 .h：需要显式 inline
类外写函数体且放在 .cpp：不用 inline
```

---

## 7. 内联展开和不展开时的调用方式

### 7.1 不内联展开：正常函数调用

C++ 代码：

```cpp
int add(int a, int b) {
    return a + b;
}

int main() {
    int x = add(1, 2);
}
```

如果不展开，生成的机器码大致类似：

```asm
main:
    mov edi, 1
    mov esi, 2
    call add
    mov [x], eax

add:
    lea eax, [rdi+rsi]
    ret
```

核心是：

```asm
call add
ret
```

`call` 会跳转到函数入口，并保存返回地址。

`ret` 会根据返回地址跳回调用点。

---

### 7.2 内联展开：函数体直接放到调用点

```cpp
inline int add(int a, int b) {
    return a + b;
}

int main() {
    int x = add(1, 2);
}
```

如果编译器决定展开，大致相当于：

```cpp
int main() {
    int x = 1 + 2;
}
```

甚至进一步优化成：

```cpp
int main() {
    int x = 3;
}
```

这时就没有这一次函数调用，也没有对应的 `call/ret`。

---

### 7.3 不展开的成本

普通函数调用通常涉及：

```text
参数传递
保存返回地址
跳转到函数入口
建立/恢复栈帧
保存/恢复部分寄存器
ret 返回
可能影响 CPU 分支预测和指令缓存
```

对于很短的函数，这些调用开销可能比函数体本身还明显。

---

### 7.4 展开不一定更快

如果函数体很大，并且在很多地方都被展开，会导致：

```text
代码体积变大
指令缓存压力变大
I-cache miss 增加
整体性能可能反而下降
```

所以编译器会权衡：

```text
展开节省函数调用成本
但可能导致代码膨胀
```

---

## 8. 函数对象

函数对象，也叫 functor，本质是一个重载了 `operator()` 的对象。

```cpp
struct Add {
    int operator()(int a, int b) const {
        return a + b;
    }
};

Add add;
int x = add(1, 2);
```

这里：

```cpp
add(1, 2);
```

实际调用的是：

```cpp
add.operator()(1, 2);
```

---

### 8.1 函数对象的核心价值

函数对象最大的价值是：

```text
像函数一样调用
但可以保存状态
```

普通函数无法自然保存状态：

```cpp
bool greaterThan10(int x) {
    return x > 10;
}
```

如果阈值是运行时传入的，函数对象更合适：

```cpp
struct GreaterThan {
    int threshold;

    GreaterThan(int t) : threshold(t) {}

    bool operator()(int x) const {
        return x > threshold;
    }
};

GreaterThan gt(100);

gt(120);  // true
gt(80);   // false
```

---

### 8.2 函数对象常用于 STL 算法

```cpp
struct Desc {
    bool operator()(int a, int b) const {
        return a > b;
    }
};

std::vector<int> v = {3, 1, 2};

std::sort(v.begin(), v.end(), Desc{});
```

等价于使用 lambda：

```cpp
std::sort(v.begin(), v.end(), [](int a, int b) {
    return a > b;
});
```

---

### 8.3 函数对象适合复杂复用逻辑

例如订单排序：

```cpp
struct Order {
    int price;
    int time;
};
```

买单排序规则：

```text
价格高优先
价格相同，时间早优先
```

可以写成：

```cpp
struct BuyOrderCompare {
    bool operator()(const Order& a, const Order& b) const {
        if (a.price != b.price) {
            return a.price > b.price;
        }
        return a.time < b.time;
    }
};
```

使用：

```cpp
std::sort(orders.begin(), orders.end(), BuyOrderCompare{});
```

或者用于优先队列：

```cpp
std::priority_queue<Order, std::vector<Order>, BuyOrderCompare> pq;
```

这种场景下，函数对象比很长的 lambda 更清晰，因为它有名字、可复用、可测试。

---

### 8.4 函数对象可作为策略类

```cpp
template <typename Filter>
void process(const std::vector<int>& nums, Filter filter) {
    for (int x : nums) {
        if (filter(x)) {
            std::cout << x << std::endl;
        }
    }
}
```

不同策略：

```cpp
struct IsEven {
    bool operator()(int x) const {
        return x % 2 == 0;
    }
};

struct GreaterThan {
    int threshold;

    GreaterThan(int t) : threshold(t) {}

    bool operator()(int x) const {
        return x > threshold;
    }
};
```

调用：

```cpp
process(nums, IsEven{});
process(nums, GreaterThan{100});
```

---

### 8.5 函数对象通常比 std::function 更高效

模板写法：

```cpp
template <typename F>
void call(F f) {
    f();
}
```

传入函数对象：

```cpp
struct Task {
    void operator()() const {
        std::cout << "run task\n";
    }
};

call(Task{});
```

这种方式在编译期确定类型，编译器更容易内联。

相比之下：

```cpp
void call(std::function<void()> f) {
    f();
}
```

`std::function` 更灵活，但通常有额外成本：

```text
类型擦除
间接调用
可能堆分配
不容易内联
```

## 9. lambda 表达式

lambda 是 C++11 引入的匿名函数对象。

基本形式：

```cpp
[capture](parameters) -> return_type {
    body
};
```

例如：

```cpp
auto f = [](int a, int b) {
    return a + b;
};

int x = f(1, 2);
```

---

### 9.1 lambda 的本质

lambda 本质上是编译器生成的匿名函数对象。

```cpp
auto f = [](int x) {
    return x + 1;
};
```

可以近似理解为：

```cpp
struct Lambda {
    int operator()(int x) const {
        return x + 1;
    }
};

Lambda f;
```

所以：

```text
lambda = 匿名函数对象
```

---

## 10. lambda 捕获的本质

lambda 捕获的外部变量，可以理解为保存进匿名函数对象的成员变量。

---

### 10.1 值捕获：成员变量是副本

```cpp
int base = 10;

auto f = [base](int x) {
    return base + x;
};
```

可以近似理解为：

```cpp
class __Lambda {
private:
    int base;

public:
    __Lambda(int b) : base(b) {}

    int operator()(int x) const {
        return base + x;
    }
};

auto f = __Lambda(base);
```

外部变量后续变化不会影响 lambda 内部副本：

```cpp
int a = 10;

auto f = [a]() {
    std::cout << a << std::endl;
};

a = 20;

f();  // 10
```

---

### 10.2 引用捕获：成员变量类似引用或指针

```cpp
int a = 10;

auto f = [&a]() {
    std::cout << a << std::endl;
};

a = 20;

f();  // 20
```

可以近似理解为：

```cpp
class __Lambda {
private:
    int& a;

public:
    __Lambda(int& ref) : a(ref) {}

    void operator()() const {
        std::cout << a << std::endl;
    }
};
```

引用捕获有生命周期风险：

```cpp
std::function<void()> f;

{
    int a = 10;
    f = [&a]() {
        std::cout << a << std::endl;
    };
}

f();  // 危险：a 已经销毁，悬垂引用
```

---

### 10.3 无捕获 lambda

```cpp
auto f = [](int x) {
    return x + 1;
};
```

大致类似：

```cpp
class __Lambda {
public:
    int operator()(int x) const {
        return x + 1;
    }
};
```

无捕获 lambda 没有状态，通常可以转换成函数指针：

```cpp
int (*fp)(int) = [](int x) {
    return x + 1;
};
```

有捕获 lambda 不可以转换为函数指针：

```cpp
int base = 10;

int (*fp)(int) = [base](int x) {
    return base + x;
};  // 错误
```

因为有捕获 lambda 需要一个对象保存状态，不能单纯表示为函数地址。

---

### 10.4 `[=]` 和 `[&]`

值捕获：

```cpp
int a = 1;
int b = 2;

auto f = [=]() {
    return a + b;
};
```

可以理解为：

```cpp
class __Lambda {
private:
    int a;
    int b;

public:
    __Lambda(int a_, int b_) : a(a_), b(b_) {}

    int operator()() const {
        return a + b;
    }
};
```

引用捕获：

```cpp
auto f = [&]() {
    return a + b;
};
```

可以理解为：

```cpp
class __Lambda {
private:
    int& a;
    int& b;

public:
    __Lambda(int& a_, int& b_) : a(a_), b(b_) {}

    int operator()() const {
        return a + b;
    }
};
```

注意：

```text
[=] 和 [&] 并不是捕获当前作用域所有变量
而是只捕获 lambda 函数体中实际使用到的外部变量
```

---

### 10.5 mutable

默认情况下，lambda 的 `operator()` 是 `const`。

```cpp
int a = 10;

auto f = [a]() {
    // a++;  // 错误
};
```

可以理解为：

```cpp
class __Lambda {
private:
    int a;

public:
    void operator()() const {
        // a++;  // 错误，因为 operator() 是 const
    }
};
```

加上 `mutable`：

```cpp
int a = 10;

auto f = [a]() mutable {
    a++;
    std::cout << a << std::endl;
};

f();  // 11
f();  // 12

std::cout << a << std::endl;  // 10
```

可以理解为：

```cpp
class __Lambda {
private:
    int a;

public:
    void operator()() {
        a++;
    }
};
```

`mutable` 允许修改 lambda 内部保存的副本，不会修改外部变量。

---

### 10.6 捕获对象成员变量时的风险

```cpp
class A {
public:
    int x = 10;

    auto getFunc() {
        return [=]() {
            return x;
        };
    }
};
```

这里很多人以为 `[=]` 是把成员变量 `x` 拷贝进 lambda。

但在 C++11/14/17 中，通常可以理解为捕获了 `this` 指针，而不是直接捕获 `x` 的副本。

近似理解：

```cpp
class __Lambda {
private:
    A* this_;

public:
    __Lambda(A* p) : this_(p) {}

    int operator()() const {
        return this_->x;
    }
};
```

如果对象已经销毁，再调用这个 lambda，就会有风险。

更安全的写法是显式拷贝成员：

```cpp
auto getFunc() {
    int copy = x;
    return [copy]() {
        return copy;
    };
}
```

或者使用初始化捕获：

```cpp
auto getFunc() {
    return [x = this->x]() {
        return x;
    };
}
```

## 11. std::function

`std::function` 是 C++ 标准库提供的通用函数包装器。

头文件：

```cpp
#include <functional>
```

它可以保存：

```text
普通函数
函数指针
lambda
函数对象
std::bind 结果
成员函数包装结果
```

---

### 11.1 基本用法

```cpp
#include <functional>
#include <iostream>

int add(int a, int b) {
    return a + b;
}

int main() {
    std::function<int(int, int)> f = add;

    std::cout << f(1, 2) << std::endl;
}
```

语法：

```cpp
std::function<返回值类型(参数类型...)>
```

例如：

```cpp
std::function<void()> f1;
std::function<int(int)> f2;
std::function<double(double, double)> f3;
```

---

### 11.2 std::function 可以保存有捕获 lambda

```cpp
int base = 10;

std::function<int(int)> f = [base](int x) {
    return base + x;
};

std::cout << f(5);  // 15
```

函数指针不能保存有捕获 lambda，但 `std::function` 可以。

---

### 11.3 std::function 的优点

`std::function` 的主要优点是统一接口。

```cpp
void registerCallback(std::function<void(int)> cb) {
    cb(100);
}
```

调用方可以传入普通函数、lambda、函数对象等：

```cpp
registerCallback([](int x) {
    std::cout << x << std::endl;
});
```

---

### 11.4 std::function 的缺点

`std::function` 比函数指针、lambda、函数对象更灵活，但也更重。

可能存在：

```text
类型擦除开销
间接调用开销
小对象优化失败时的堆分配
不利于内联
```

所以在高性能场景，尤其是热路径代码中，不应无脑使用 `std::function`。

可以优先考虑：

```text
模板参数
函数指针
auto / lambda 直接传递
函数对象
```

---

## 12. 函数对象、lambda、std::function 的关系

### 12.1 lambda 是匿名函数对象

这个 lambda：

```cpp
int base = 10;

auto f = [base](int x) {
    return base + x;
};
```

大致等价于：

```cpp
struct LambdaLike {
    int base;

    LambdaLike(int b) : base(b) {}

    int operator()(int x) const {
        return base + x;
    }
};

auto f = LambdaLike{base};
```

所以：

```text
lambda 是编译器生成的匿名函数对象
函数对象是显式写出来的可调用对象
std::function 是通用可调用对象包装器
```

---

### 12.2 对比表

| 类型            | 能否保存状态 | 调用开销 | 灵活性 | 典型用途         |
| ------------- | -----: | ---: | --: | ------------ |
| 普通函数          |      否 |    低 |   低 | 基础工具函数       |
| 函数指针          |      否 |    低 |   中 | C 风格回调、简单策略  |
| inline 函数     |  看函数本身 | 可能最低 |   中 | 短小工具函数、头文件函数 |
| 函数对象          |     可以 |  通常低 |   高 | 策略类、比较器、过滤器  |
| lambda        |     可以 |  通常低 |   高 | 临时逻辑、回调、排序   |
| std::function |     可以 |   较高 |  最高 | 通用回调接口、事件系统  |

---

## 13. 回调接口设计对比

### 13.1 函数指针版本

```cpp
void registerCallback(void (*cb)(int)) {
    cb(1);
}
```

可以接普通函数或无捕获 lambda：

```cpp
registerCallback([](int x) {
    std::cout << x << std::endl;
});
```

不能接有捕获 lambda：

```cpp
int base = 10;

registerCallback([base](int x) {
    std::cout << base + x << std::endl;
});  // 错误
```

---

### 13.2 std::function 版本

```cpp
void registerCallback(std::function<void(int)> cb) {
    cb(1);
}
```

可以接有捕获 lambda：

```cpp
int base = 10;

registerCallback([base](int x) {
    std::cout << base + x << std::endl;
});
```

优点是灵活，缺点是有类型擦除和间接调用开销。

---

### 13.3 模板版本

```cpp
template <typename Callback>
void registerCallback(Callback cb) {
    cb(1);
}
```

可以接各种 callable：

```cpp
registerCallback([](int x) {
    std::cout << x << std::endl;
});
```

优点：

```text
性能好
容易内联
没有 std::function 类型擦除开销
```

缺点：

```text
通常要放在头文件
可能增加编译时间
错误信息更复杂
接口不如 std::function 明确
```

## 14. 面试常见问题

### 14.1 inline 是否一定内联展开？

不是。

`inline` 在语言层面主要表示：

```text
允许函数定义出现在多个翻译单元中
```

是否真的展开，由编译器优化决定。

---

### 14.2 类内定义的成员函数是否默认 inline？

是。

```cpp
class A {
public:
    void f() {
        // ...
    }
};
```

这种类内定义的成员函数隐式是 `inline`。

---

### 14.3 `.h` 里声明的函数都是 inline 吗？

不是。

只有类内直接写函数体，才是隐式 `inline`。

```cpp
class A {
public:
    void f();  // 只是声明，不是 inline
};
```

---

### 14.4 有捕获 lambda 能转换成函数指针吗？

不能。

```cpp
int base = 10;

int (*fp)(int) = [base](int x) {
    return base + x;
};  // 错误
```

因为有捕获 lambda 需要对象保存状态。

---

### 14.5 无捕获 lambda 能转换成函数指针吗？

可以。

```cpp
int (*fp)(int) = [](int x) {
    return x + 1;
};
```

---

### 14.6 lambda 捕获的变量存在哪里？

可以理解为存放在 lambda 对象内部，作为成员变量。

```text
值捕获：保存副本
引用捕获：保存引用或地址
```

---

### 14.7 mutable 的作用是什么？

`mutable` 让 lambda 的 `operator()` 不再是 `const`，因此可以修改值捕获产生的内部副本。

```cpp
int a = 10;

auto f = [a]() mutable {
    a++;
};
```

这里修改的是 lambda 内部的副本，不是外部的 `a`。

---

### 14.8 std::function 为什么可能比 lambda 慢？

因为 `std::function` 通常涉及：

```text
类型擦除
间接调用
可能堆分配
不容易内联
```

而 lambda 或函数对象通过模板传递时，类型在编译期确定，更容易被优化。

---

### 14.9 成员函数指针和普通函数指针一样吗？

不一样。

普通函数指针：

```cpp
void (*)();
```

成员函数指针：

```cpp
void (A::*)();
```

普通成员函数调用时需要对象，也就是隐藏的 `this` 指针。

---

## 15. 总结

可以按抽象层次理解：

```text
普通函数：一段固定代码
函数指针：保存普通函数地址
函数对象：可以像函数一样调用的对象
lambda：编译器生成的匿名函数对象
std::function：统一包装各种可调用对象
inline：影响函数定义规则和可能的调用展开
```

从性能角度看：

```text
普通函数 / 函数指针：轻量，但表达能力弱
函数对象 / lambda：表达能力强，性能通常也好
std::function：最灵活，但有额外开销
inline：不保证展开，是否展开由编译器决定
```

最核心的几句话：

```text
1. inline 保证的是多翻译单元中相同定义合法，不保证一定展开。
2. 类内定义的成员函数隐式 inline，类内声明不是 inline。
3. 不展开时通过 call/ret 调用函数；展开时函数体直接放到调用点。
4. lambda 本质是匿名函数对象。
5. lambda 捕获的外部变量可以理解为匿名函数对象的成员变量。
6. 值捕获保存副本，引用捕获保存引用或地址。
7. 函数对象的价值是既能像函数一样调用，又能保存状态。
8. std::function 最灵活，但可能有类型擦除、间接调用和堆分配开销。
```

## 16. 面试回答模板

如果面试官问：

> C++ 里普通函数、函数指针、lambda、函数对象、std::function 有什么区别？

可以这样回答：

> 普通函数是最基础的可调用实体，函数名在很多场景下可以退化为函数地址。函数指针保存的是函数入口地址，只能指向普通函数、static 成员函数或无捕获 lambda，调用开销低，但不能保存状态。函数对象是重载了 `operator()` 的对象，可以像函数一样调用，同时保存状态，常用于 STL 比较器、谓词和策略类。lambda 本质上是编译器生成的匿名函数对象，捕获的外部变量可以理解为这个匿名对象的成员变量。`std::function` 是通用函数包装器，可以统一保存普通函数、函数指针、lambda、函数对象等 callable，但由于类型擦除、间接调用和可能的堆分配，性能上通常比直接使用 lambda 或函数对象更重。`inline` 不是一种函数类型，它主要表示函数可以在多个翻译单元中有相同定义，是否真的内联展开由编译器优化决定。
